# Repository Guidelines

## Project Structure & Module Organization

BinoCart is a standalone static web and Android/Capacitor prototype.

- `index.html` is the Market Intel/Pulse page.
- `scan.html`, `list.html`, `history.html`, `saved.html`, `receipts.html`, and `profile.html` are separate app views.
- `app.js` contains scan, lookup, Pulse, cart, product-detail, and Intel/Scan logic.
- `pages.js` contains shared page behavior for lists, history, saved products, receipt review, profile receipt memory, and price memory graphs.
- `styles.css` contains app-wide responsive styling.
- `scripts/` contains the local static server, receipt API, SQLite receipt DB worker, web build, Capacitor, and Android helpers.
- `tests/` contains Node `node:test` coverage.
- `assets/brand/` contains BinoCart brand assets and references.
- `www/` is generated by `npm run build:web`; do not hand-edit it.

## Build, Test, and Development Commands

Install dependencies:

```powershell
npm install
```

Run the app locally:

```powershell
npm run dev:web
```

Run the receipt parser API:

```powershell
npm run api:receipts
```

Run tests:

```powershell
npm test
```

Build the Capacitor web bundle:

```powershell
npm run build:web
```

Do not run `npm test` and `npm run build:web` at the same time because the build rewrites `www/`.

## Coding Style & Naming Conventions

Use plain HTML, CSS, and JavaScript. Keep indentation consistent with nearby files, usually two spaces. Use descriptive camelCase for JavaScript names and kebab-case for CSS classes and asset filenames.

UI copy should stay concise, explainable, and customer-first. BinoCart should feel like independent shopping intelligence, not a retailer or marketplace.

## Testing Guidelines

Use Node's built-in `node:test` runner. Add or update tests when changing Pulse/cart logic, receipt parsing/review, localStorage profile behavior, static page wiring, or the SQLite receipt DB worker. Manually verify camera scanning on HTTPS-capable mobile browsers when scanner code changes.

## Security & Local Data

Do not commit `.env`, `.env.*`, `data/`, `node_modules/`, `www/`, `android/local.properties`, or Android build output. OpenAI credentials must stay in `.env` or server environment variables and must never be added to browser code.

## Commit & Push Notes

Use short imperative commit messages, such as `Add receipt memory database and review UI`. When pushing changes, include the app test links in the final response:

- Local: `http://127.0.0.1:4173/index.html`
- GitHub Pages: `https://ajth-work.github.io/binocart/`
- Actions: `https://github.com/ajth-work/binocart/actions`
